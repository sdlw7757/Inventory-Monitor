## 🚀 快速开始

1. 下载项目，双击打开 `index.html`
2. 配置 QQ 邮箱 SMTP 授权码
3. 设置接收短信手机号
4. 添加监控网址、商品关键词、提醒阈值
5. 点击 **启动监控** 即可

### 2. 云端 24 小时运行（GitHub）

1. 将项目上传到你的 GitHub 仓库
2. 在仓库配置 3 个密钥：
   - `FROM_EMAIL`：发件 QQ 邮箱
   - `FROM_PWD`：QQ 邮箱 SMTP 授权码
   - `TO_PHONE`：接收短信手机号
3. 自动开始监控，无需开机

## 📱 使用说明

### 1. 短信提醒配置

- **发件邮箱**：填写你的 QQ 邮箱
- **授权码**：QQ 邮箱 → 设置 → 账户 → 开启 SMTP → 获取授权码
- 保存后输入框自动清空，显示脱敏邮箱
- 支持删除配置，重置信息

### 2. 接收手机号设置

- 填写国内 11 位手机号
- 保存后自动脱敏显示：135\*\*\*\*6789
- 支持删除，关闭短信通知

### 3. 添加监控项

- **监控网址**：商品页面完整链接
- **商品关键词**：商品唯一标识（如 `16H-16G-HKTJ-003`）
- **阈值**：库存大于该数值时发送提醒

### 4. 启动 / 停止监控

- 启动后每 5 分钟检测一次库存
- 支持随时停止
- 实时日志查看运行状态

### 5. 📄 config.json 配置文件详细说明

**作用**

`config.json` 是 GitHub 云端监控的唯一配置文件，云端脚本会读取该文件，知道要监控哪些商品。

**生成方式**

1. 打开 `index.html`
2. 添加好所有监控项
3. 点击按钮 **【导出云端配置（config.json）】**
4. 浏览器会自动下载 `config.json`
5. 将该文件放到项目根目录，一起上传到 GitHub

**文件格式示例**

```json
[
  {
    "url": "https://example.com/product1",
    "keyword": "4H-4G-USTJ-002",
    "threshold": 0
  },
  {
    "url": "https://example.com/product2",
    "keyword": "16H-16G-HKTJ-003",
    "threshold": 0
  }
]
```

**字段说明**

| 字段 | 说明 |
|------|------|
| url | 商品页面的完整网址，必须能公开访问 |
| keyword | 商品唯一关键词，用于定位库存位置 |
| threshold | 提醒阈值，库存 > 该值时触发短信告警 |

**注意事项**

- 必须导出后再上传，不要手动手写，避免格式错误
- 修改监控项请在网页操作 → 重新导出 → 覆盖上传
- 云端监控只认 `config.json`，不同步网页本地配置

## 🛠️ 常见问题

### Q1：获取不到 QQ 邮箱授权码？

进入 QQ 邮箱 → 设置 → 账户 → POP3/IMAP 服务 → 开启 → 获取授权码

### Q2：本地收不到短信？

浏览器安全限制：本地仅做演示，真实短信必须在 GitHub 云端运行

### Q3：库存抓取不准确？

- 关键词必须是商品唯一标识
- 不要使用重复、通用词语
- 导出配置前确保本地抓取正常

### Q4：GitHub 不运行？

检查：
- 配置文件路径是否正确 `.github/workflows/monitor.yml`
- 密钥名称是否正确（区分大小写）
- `config.json` 是否存在
